--介绍--
PO的基本框架，包括以下部分;
框架用的主要技术栈是：Python + appium+ unittest + unittestreport + Jenkins，考虑到ui自动化的维护成本，我们采用了POM的设计模式，就是将业务流程上涉及的页面，封装装为一个页面对象。
目录如下：
--1.conf目录：用于存放配置文件
--2.data目录：用于存放测试用例数据的文件，我们之前的项目使用的是yaml的文件进行测试用例数据的管理，每一个测试用例脚本对应一个yaml文件，每一个yaml文件中可能有多组测试用例数据，用于测试不同的业务场景
--3.page目录：用于封装页面对象，将页面封装为页面对象类，将页面的元素封装为类的属性，将页面的操作行为封装为类下的实例方法；
--4.public目录：用于封装公共的函数和类，用于提供token等操作，用于测试脚本来调用
--5.testcases目录：用于存放测试用例的脚本，我们使用POM模式将业务的流程与测试用例脚本进行分离；
--6.report目录：用于存放测试报告的目录，我们使用allure生成测试报告
--7.run.py文件：这是整个框架的一键执行脚本文件，用于日常的执行及要执行用例的过滤
